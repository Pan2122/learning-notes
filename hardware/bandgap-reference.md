# 带隙基准原理与芯片测试学习指南

> 简介：从“为什么芯片需要一个稳定电压”出发，梳理带隙基准的 CTAT/PTAT 原理、关键规格、测试路径、实验室表征、ATE 量产测试和异常定位方法。

## 如何使用这份材料

建议按三轮学习：第一轮建立直觉，第二轮看公式与指标，第三轮只看测试表格和复习题。带隙基准涉及器件物理、模拟电路和测试工程，不建议一上来死记电路。

| 学习轮次 | 重点 | 达到的目标 |
| --- | --- | --- |
| 第一轮 | 第 1～4 章 | 能用自己的话解释 CTAT + PTAT，知道为什么参考电压约为 1.2 V。 |
| 第二轮 | 第 5～8 章 | 能看懂 VREF(T)、ppm/°C、line regulation、PSRR、noise、startup 等指标。 |
| 第三轮 | 第 9～13 章 | 能设计测试矩阵、识别测量陷阱，并根据异常形状定位问题。 |

### 学习地图

先问：芯片为什么需要“内部标尺”？

再看：VBE 随温度下降，ΔVBE 随绝对温度上升。

把二者按比例相加，获得一阶温度系数接近零的参考。

理解真实 BGR 还包含启动、电流偏置、运放、trim、buffer 和测试 MUX。

最后把精度拆成初始误差、温漂、供电敏感度、负载、噪声、启动、老化与封装应力。

> 最先记住的一句话：带隙基准不是“永远等于 1.2 V 的电源”，而是利用两种相反温度特性的电压，合成一个尽量不随温度和电源变化的内部参考。

## 1. 从“尺子”开始理解带隙基准

### 1.1 芯片为什么需要参考电压

数字逻辑只需区分 0 和 1；模拟电路却必须回答“这个电压到底是多少”。如果芯片内部没有一个相对稳定的基准，ADC 的满量程、LDO 的输出、比较器阈值、传感器的偏置电流都会随电源、温度和工艺漂移。

> 直觉类比 VDD 像一根会热胀冷缩的尺子；带隙基准像芯片内部校准过的标准尺。它不一定绝对完美，但要比被测对象稳定得多。

![带隙基准在芯片中的共享参考路径](/images/hardware/bandgap-reference/reference-architecture.png)

*图 1 带隙基准常被多个模块共享；因此它的误差会向下游扩散。*

### 1.2 它和稳压器有什么区别

| 对象 | 主要任务 | 典型关注点 |
| --- | --- | --- |
| 带隙基准 / Voltage Reference | 提供准确、稳定的“标尺电压”或基准电流 | 精度、温漂、噪声、PSRR、启动、老化 |
| LDO / Regulator | 给负载供电，并承受较大输出电流 | 负载能力、压差、环路稳定性、瞬态、效率 |
| 电源 VDD | 系统提供的能量来源 | 范围、纹波、瞬态、上电斜率 |

很多 LDO 的误差放大器会把“分压后的输出”与内部 VREF 比较，因此 LDO 最终输出常可近似写成：

> **公式**：`VOUT = G × VREF`
>
> G 由反馈电阻比或内部增益决定。VREF 偏 0.5%，理想情况下 VOUT 也会偏约 0.5%。

## 2. “带隙”这个名字到底是什么意思

### 2.1 先排除一个常见混淆

带隙基准（bandgap reference）与 SiC、GaN 等“宽禁带半导体”不是同一个话题。二者都使用 bandgap 这个词，但前者是一种参考电路，后者描述材料的能带特性。

### 2.2 为什么最后约为 1.2 V

在经典 BJT 温度模型中，把 VBE(T) 外推到 0 K 会得到一个与材料相关的常数 VG0，工程上常取约 1.2 V。经典带隙电路把一个具有负温度系数的结电压与一个具有正温度系数的电压相加，在一阶近似下，合成值落在 VG0 附近，因此被称为 bandgap reference。

> 重要但容易说错：“硅在室温的材料带隙约 1.12 eV”“零温材料带隙约 1.17 eV”与“经典 BGR 输出约 1.2 V”不能机械画等号。BGR 中常说的约 1.205 V 是 VBE 温度模型的外推常数 VG0/工程近似，不是简单把室温带隙能量除以电子电荷。

真实芯片的输出不一定就是 1.2 V。低电压工艺会使用 sub-bandgap、current-mode 或 scaled reference；对外可见的参考还可能经过分压、放大或 buffer，成为 0.6 V、0.9 V、1.5 V、2.5 V 等数值。

## 3. 核心原理：CTAT + PTAT

### 3.1 CTAT：VBE 随温度升高而下降

双极晶体管的基极—发射极电压 VBE 在固定电流条件下具有负温度系数，常见数量级约为 -1.5～-2.2 mV/°C，具体数值取决于电流密度、工艺和工作点。这类量称为 CTAT（Complementary To Absolute Temperature）。

### 3.2 PTAT：ΔVBE 随绝对温度上升

让两个匹配 BJT 工作在不同电流密度下，它们 VBE 的差值为：

> **公式**：`ΔVBE = VT × ln(N) = (kT/q) × ln(N)`
>
> N 是电流密度比，VT = kT/q 为热电压；在 25°C 时 VT 约为 25.7 mV。

因为 ΔVBE 与绝对温度 T 成正比，所以它是 PTAT（Proportional To Absolute Temperature）量。电路通常用电阻把 ΔVBE 转成 PTAT 电流，再通过另一个电阻产生放大后的 PTAT 电压。

### 3.3 一阶相消

> **公式**：`VREF = VBE + K × ΔVBE`
>
> 设计 K，使 dVREF/dT = dVBE/dT + K·d(ΔVBE)/dT ≈ 0。

![CTAT 与 PTAT 一阶温度补偿示意](/images/hardware/bandgap-reference/ctat-ptat-compensation.png)

*图 2 CTAT 与 PTAT 一阶相消示意。相消后仍会留下曲率。*

> 关键认知：带隙基准并不是找到一个天然不随温度变化的器件，而是把两个“都会变、但变化方向相反”的量按比例组合。

## 4. 用一个数字例子把公式算通

假设两个 BJT 的电流密度比 N = 8，温度为 25°C（约 298 K）。

热电压 VT = kT/q ≈ 25.7 mV。

ΔVBE = 25.7 mV × ln(8) ≈ 53.4 mV。

若 VBE 的温度斜率约为 -1.85 mV/°C，则 ΔVBE 的斜率约为 (k/q)ln(8) ≈ 0.179 mV/°C。

需要的放大系数 K 约为 1.85 / 0.179 ≈ 10.3。

若此工作点 VBE 约 0.66 V，则 VREF ≈ 0.66 + 10.3×0.0534 ≈ 1.21 V。

> 这个例子说明了什么：约 0.6～0.7 V 的 VBE 加上被放大约十倍的 ΔVBE，得到约 1.2 V；电阻比决定 K，BJT 面积比或电流比决定 N。

### 4.1 为什么温度曲线仍然不是完全水平

VBE 的温度关系不是严格直线，存在高阶曲率。

电阻本身有温度系数，且不同类型电阻的温度系数不同。

运放失调、电流镜误差、有限增益、漏电和寄生效应随温度变化。

封装应力、焊接应力和版图温度梯度会引入额外漂移。

因此高精度 BGR 常加入曲率补偿，而 trim 也未必只调一个参数。

## 5. 真实芯片里的带隙基准不只有两个 BJT

教科书公式只描述核心。真正可用的 BGR 通常还包括以下模块：

| 模块 | 作用 | 典型异常 |
| --- | --- | --- |
| 启动电路 | 打破“零电流也是平衡点”的假稳态，使 BGR 正常建立 | 某些上电斜率、低温或低 VDD 下不启动 |
| 偏置/电流镜 | 建立 PTAT/CTAT 电流并复制到其他模块 | 工艺角、VDD 和失配导致比例偏差 |
| 误差放大器 | 强制关键节点相等或建立正确电流密度比 | 失调、有限增益、输入共模范围不足 |
| 电阻网络 | 设定 K、缩放输出、提供 trim 权重 | 比值偏差、温度系数、码跳变 |
| trim/OTP/eFuse | 修正初始误差，部分架构也可修正斜率 | trim 码算错、烧写失败、码效非单调 |
| buffer | 隔离负载并提高驱动能力 | 负载调节、稳定性、输出摆幅 |
| 测试 MUX | 把内部节点路由到可测 PAD 或 ADC | MUX 电阻、漏电、PAD 保护影响测量 |

### 5.1 为什么一定要有启动电路

许多自偏置回路同时存在两个数学解：目标工作点和零电流点。若上电后停在零电流点，VREF 可能为 0 V、异常低或随机建立。启动电路只应在建立阶段注入扰动，正常工作后要尽量退出，避免给核心带来额外误差和噪声。

### 5.2 BJT 从哪里来

在双极工艺中可直接使用纵向 NPN/PNP；在 CMOS 工艺中常利用寄生 PNP 或兼容器件。器件类型会影响增益、匹配、噪声、压头和最低供电电压，因此低压 CMOS BGR 往往使用与经典 Brokaw 单元不同的拓扑。

## 6. 为什么带隙基准在芯片中这么重要

| 下游模块 | VREF 的角色 | VREF 偏差可能表现为 |
| --- | --- | --- |
| ADC / DAC | 定义满量程或 LSB 大小 | 增益误差、量程变化、不同温度下比例漂移 |
| LDO / DCDC | 反馈比较的目标值 | 输出电压整体偏高/偏低，随温度漂移 |
| MEMS / Sensor AFE | 偏置、电桥/电容激励、比较阈值、ADC 标尺 | 灵敏度、零偏、量程、阈值或温度补偿误差 |
| 温度传感器 | 把 PTAT/CTAT 量转换成数字码的参考 | 温度读数斜率或截距错误 |
| RC/Relaxation OSC | 设定充放电阈值或偏置电流 | 频率偏差和温漂 |
| POR / BOR | 电源监测比较阈值 | 过早/过晚复位，边界电压行为异常 |
| 内部 bias | 产生模拟模块所需电流 | 带宽、噪声、功耗、驱动能力整体偏移 |

### 6.1 误差如何传到 ADC

> **公式**：`Code ≈ (VIN / VREF) × FullScale`
>
> 小信号近似：δCode/Code ≈ δVIN/VIN − δVREF/VREF。若 VIN 不变，VREF 增大 0.1%，数字码约减小 0.1%。

### 6.2 对传感器芯片的特殊意义

对于加速度计、温度传感器等混合信号芯片，输出常经过“机械敏感结构/模拟前端/ADC/数字补偿”多级链路。如果 VREF 同时参与激励和 ADC，误差可能部分抵消，也可能以更复杂的形式保留下来；不能看到 ADC 码变化就直接断言 BGR 异常。

> AE/测试视角：先画清楚 VREF 进入了哪些模块，以及每条路径的传递函数。否则只能看到系统症状，无法判断参考误差是被放大、被抵消，还是根本不可观测。

## 7. 评价带隙基准的关键指标

| 指标 | 它回答的问题 | 常见表达 |
| --- | --- | --- |
| 初始精度 | 25°C、规定 VDD/负载下，单颗偏离标称多少？ | %、mV、ppm |
| 温度系数 TC | 跨温度范围变化多大？ | ppm/°C；box、endpoint 或 fit 定义 |
| 曲率 | 温漂是否呈弓形/S 形，而不是简单直线？ | VREF-T 曲线、残差 |
| Line regulation | VDD 变化时 VREF 变化多少？ | µV/V、ppm/V |
| PSRR | 电源上的交流扰动有多少耦合到 VREF？ | dB vs frequency |
| Load regulation | 输出负载变化时 VREF 变化多少？ | µV/µA、µV/mA |
| 输出阻抗 | VREF 是否足以驱动动态负载？ | Ω vs frequency |
| 噪声 | 参考自身的随机波动有多大？ | 0.1–10 Hz µVpp、nV/√Hz、µVrms |
| 启动/建立时间 | 各种上电条件下能否正确、及时建立？ | 成功率、settling time |
| 静态电流 | 参考工作消耗多少功耗？ | µA、nA |
| 迟滞/老化 | 经历温循或长期工作后是否回到原值？ | ppm、mV |

### 7.1 温度系数的定义必须先讲清

> **公式**：`TCbox = [(Vmax − Vmin) / (Vnom × (Tmax − Tmin))] × 10⁶ ppm/°C`
>
> 这是常见 box method。endpoint slope 与线性拟合得到的 TC 可能完全不同，不能混用。

> 测试报告常见错误：只给一个“TC = 20 ppm/°C”却不说明温度范围、计算方法、trim 状态和样本量，这个数字几乎无法复现。

### 7.2 PSRR 与 line regulation 不是一回事

Line regulation 是准静态 DC 供电变化；PSRR 是某一频率下的交流扰动抑制能力。一个 BGR 的 DC line regulation 很好，并不代表它在 1 MHz 开关电源纹波下仍有很高 PSRR。

> **公式**：`PSRR(dB) = 20 log10(Vripple,in / Vripple,ref)`
>
> 按正值表达时，数值越大表示抑制越好；报告中仍应注明注入幅度、频率、偏置点和测量带宽。

## 8. 为什么必须测试

工艺变化：BJT 的 IS、β、VBE，电阻绝对值与比值都会变化。

器件失配：左右支路不完全对称，运放存在输入失调，电流镜存在系统与随机误差。

温度高阶效应：一阶相消正确，并不保证整个 -40～125°C 曲线合格。

电源和压头：低 VDD、高温、低温或特定工艺角下，某些晶体管可能退出饱和区。

启动风险：不同上电斜率、掉电时间和残余电荷可能让回路卡在错误状态。

封装/焊接应力：晶圆测试与封装后测试结果可能出现系统偏移。

trim 风险：码权、算法、OTP 烧写和读回正确，不等于模拟值一定正确。

系统影响：BGR 往往是共因失效点，可能同时影响 ADC、LDO、POR 和传感器输出。

### 8.1 设计仿真为什么不能替代硅后测试

仿真可以覆盖 PVT corner、Monte Carlo 和器件模型，但真实硅片还包含模型不完备、版图寄生、梯度、封装应力、测试 MUX 和测量接口等因素。测试的目标不是“证明仿真没错”，而是量化模型与产品现实之间的差异，并建立可量产的限值。

### 8.2 表征测试与量产测试的目标不同

| 阶段 | 主要目标 | 特点 |
| --- | --- | --- |
| Characterization / EVT-DVT | 理解边界、相关性、温漂形状和失效机理 | 样本较少，但温度/VDD/频率/负载条件多，允许较长测试时间 |
| Qualification / Reliability | 确认应力前后漂移和长期稳定性 | 高温工作寿命、温循、湿热、封装应力等前后对比 |
| Production test | 快速筛除不良品并执行 trim | 测试时间严格受限，使用少数最有判别力的条件，依赖 guardband |

## 9. 测试前先解决：到底测哪个节点

![带隙基准测试观测路径选择](/images/hardware/bandgap-reference/observation-path.svg)

*图 3 直接可测与间接可观测是两条不同的测试路线。（原 Word 截图已重绘为网页友好的 SVG。）*

### 9.1 三个常被混为一谈的电压

| 名称 | 含义 | 为什么可能不同 |
| --- | --- | --- |
| VBG core | 带隙核心内部合成的电压 | 未经过 buffer、分压、放大或 trim MUX |
| VREF internal | 送给 ADC/LDO/其他模块的内部参考 | 可能经过缩放、trim、滤波或多级 buffer |
| VREF pad | 测试模式下路由到外部 PAD 的电压 | MUX 导通电阻、PAD 漏电、ESD、测试机负载会改变它 |

> 在写测试规格前：必须拿到电路框图、test mode 路由和增益关系。若测试 PAD 上测到 0.8 V，不代表核心不是 1.2 V；它可能经过 2/3 分压。

### 9.2 直接测量的基本连接

用低噪声电源给 DUT 供电，设置 VDD、EN、test mode 和 trim 状态。

将内部 VREF 通过 test MUX 路由到模拟 PAD。

用高输入阻抗 DMM、SMU sense 或 ATE digitizer 测量，避免测试机加载。

设置足够的等待时间，区分 enable 到电压建立、继电器稳定和仪器 aperture。

同步记录 VDD、IDD、温度、trim code、lot/wafer/site 与测量时间戳。

### 9.3 无法直接引出时怎样间接测

ADC 比值法：施加已知 VIN，利用 Code ≈ VIN/VREF×FS 反推 VREF。

比较器阈值扫描：扫描外部电压，找到翻转点，再除去已知分压/增益。

LDO 输出反推：若 VOUT = G×VREF 且 G 精确可知，可用 VOUT 间接判断。

POR/BOR 扫描：缓慢扫描 VDD，测复位释放/触发阈值，但要分离迟滞和斜率影响。

内部 monitor code：读取芯片内部 ADC 对 VBG/VDD 的监测码，但需先验证数字路径与参考的相关性。

> 间接测试的本质：不是“找一个相关输出看一眼”，而是建立可验证的传递函数，并把其他误差源计入测量不确定度。

## 10. 实验室与表征测试：逐项怎么测

### 10.1 室温初始精度

> Initial error(%) = (Vmeas − Vnom) / Vnom × 100% 也可乘 10⁶ 表示 ppm。必须注明 VDD、负载、温度、等待时间、trim 状态和测量节点。

先验证仪器准确度、线缆漏电、板级压降和 test MUX 增益。

多次上电测量，避免把偶发启动失败平均成一个“偏低电压”。

先看未 trim 分布，再看 trim 后分布；不要只报告最终平均值。

### 10.2 温度扫描与温漂

选择覆盖规格的温度点，例如 -40、-20、25、60、85、105、125°C；精细表征可增加点数。

每个温度点充分 soak，并确认结温而不只是箱体空气温度。

在每个温度点测 VREF，同时记录 VDD、IDD、环境温度和必要的片上温度码。

至少绘制每颗样品的 VREF-T 曲线；同时看绝对电压和相对 25°C 的偏移。

分别计算 box TC、endpoint slope 或拟合残差，名称不能混用。

必要时做升温与降温两条曲线，观察热迟滞和封装应力。

### 10.3 Line regulation

> **公式**：`Line regulation = ΔVREF / ΔVDD`
>
> 常用单位 µV/V 或 ppm/V。测试时保持温度、负载和模式不变，并覆盖最低工作压头附近。

建议先做慢速 VDD sweep，观察是否存在线性斜率、拐点或突然失调。若只测 VDDmin 与 VDDmax 两点，可能错过内部器件退出工作区造成的中间拐点。

### 10.4 Load regulation 与输出能力

只有在被测节点包含 buffer 且规格允许外部负载时才有意义。对裸 BGR 核心 PAD 施加大电流，可能测到的是 MUX 与保护结构，而不是 BGR 本身。

> **公式**：`Load regulation = ΔVREF / ΔILOAD`
>
> 同时观察输出稳定性、settling、振铃和是否触发限流。动态负载时还需关注输出电容与 ESR。

### 10.5 PSRR

在 DC VDD 上叠加小信号正弦纹波，幅度要足够可测但不能让电路进入非线性。

扫描频率，例如 10 Hz 到 10 MHz；实际范围取决于产品和测试板带宽。

分别测电源纹波幅度 Vin,ripple 与参考端纹波 Vref,ripple。

计算 PSRR，并验证夹具、去耦、电源注入网络和示波器底噪没有主导结果。

对低频可用锁相/FFT；对高频注意探头接地、电容耦合和 PAD 寄生。

### 10.6 噪声

低频噪声常报告 0.1–10 Hz 峰峰值；宽带噪声常报告噪声密度 nV/√Hz 或积分 RMS。

使用电池或低噪声电源、屏蔽盒、低噪声前置放大器和合理带宽。

先短接输入测系统底噪，再接 DUT；若仪器底噪接近 DUT，结果只能给上限。

避免把 50/60 Hz、开关电源纹波、数字活动或温漂误当成 BGR 本征噪声。

噪声测试不能简单用普通 DMM 读数的最大值减最小值代替。

### 10.7 启动与上电

| 变量 | 建议覆盖 | 观察量 |
| --- | --- | --- |
| VDD 上升斜率 | 很慢、典型、很快 | 是否启动、过冲、settling |
| 起始电压 | 0 V、brownout 区间、非完全掉电 | 是否卡在假稳态 |
| 掉电时间 | 短间隔到充分放电 | 残余电荷影响 |
| 温度 | 低温、室温、高温 | 器件 gm、漏电和启动裕量 |
| 负载/电容 | 最小、典型、最大允许值 | 建立时间和稳定性 |
| 重复次数 | 至少数十次；风险项可更多 | 偶发失败概率 |

> 启动测试的判定：不仅要看最终电压是否正确，还要定义在多长时间内进入多大的误差带，例如 enable 后 500 µs 内进入 Vnom±0.5%。

## 11. Trim：测什么，不能期待什么

### 11.1 一点 trim 与两点 trim

| 方法 | 可主要修正 | 局限 |
| --- | --- | --- |
| 室温一点 trim | 25°C 初始偏差/截距 | 不能保证温度斜率与曲率同时变好 |
| 两温点 trim | 截距 + 一阶温度斜率 | 测试时间和温控成本高，仍可能残留曲率 |
| 多参数/曲率 trim | 截距、斜率甚至高阶项 | 电路与算法复杂，码间耦合需验证 |

### 11.2 一个完整的 trim 流程

测量 pre-trim VREF，确认 DUT 已正常启动。

根据灵敏度或 trim transfer curve 计算候选码。

若允许，先用 shadow register/scan 写入候选码并复测。

选择使目标误差最小且留有温度/量产裕量的码。

烧写 OTP/eFuse，并执行读回校验。

重新上电，测 post-trim VREF；必要时再测关键温度点。

保存 pre/post 值、最终码、码步进、烧写状态和异常标志。

### 11.3 Trim 验证必须关注

码效是否单调，是否有缺码、反码或权重接错。

码步进是否符合设计，目标是否落在可 trim 范围内。

不同 VDD、温度和模式下 trim 码是否仍生效。

OTP 读回正确是否真的改变了模拟开关状态。

trim 后平均值变准的同时，sigma 是否异常扩大。

极端 pre-trim 样品被推到边缘码时，是否还有 guardband。

## 12. ATE 量产测试如何取舍

量产测试不能照搬完整表征矩阵。核心问题是：用尽可能少的时间，筛出会影响产品规格的 BGR 异常，并保证 trim 可追溯。

### 12.1 一个常见的生产测试骨架

Continuity / leakage：先排除 PAD 与供电硬故障。

IDD / basic startup：确认参考回路已建立且电流合理。

Pre-trim VREF：判断是否在可 trim 窗口内。

计算并加载 trim code：优先 shadow 试写。

OTP/eFuse program + verify：确认烧写和读回。

Post-trim VREF：执行最终上下限判定。

可选 hot/cold screen 或间接系统测试：覆盖温漂/启动的高风险项。

Data logging：记录原始值、最终码、site、wafer/lot 与 bin。

### 12.2 测量精度与 guardband

量产限值不能只复制设计规格。需要把测试机准确度、重复性、site-to-site 偏差、夹具漏电、温度误差、MUX 比例误差和长期漂移纳入测量不确定度。

> Product limit = Specification limit − Measurement guardband。Guardband 的方向和大小应来自 MSA/GR&R、相关性数据和失效风险，而不是经验随意缩紧。

### 12.3 量产测试常见陷阱

| 现象 | 可能并非 DUT 的原因 | 检查方法 |
| --- | --- | --- |
| 所有 site 同时偏低 | 测试机校准、共享电源压降、参考地偏移 | golden unit、site swap、Kelvin sense |
| 某一 site 均值偏移 | relay、socket、通道增益或漏电 | 交叉换 site、空载/短路校验 |
| 测值随等待时间变化 | 未建立、DMM aperture、继电器热电势 | settling sweep、重复采样曲线 |
| 高温散布突然增大 | 漏电、温控不稳、MUX/PAD 污染 | 空载漏电、温度记录、直接/间接相关性 |
| trim 后 bimodal | 码算法边界、烧写失败、码映射错误 | pre/post/码值联合散点图 |

## 13. 看曲线定位问题：症状到原因

| 异常形状/症状 | 优先怀疑 | 下一步实验 |
| --- | --- | --- |
| 全温度近似平行上移/下移 | 初始偏差、trim 截距、测试 MUX 增益 | 核对 pre/post trim、直接测 core 与 pad |
| 温度曲线明显线性倾斜 | PTAT/CTAT 权重错误、电阻比 TC、斜率 trim | 两温点拟合、扫 trim 斜率码 |
| 弓形或 S 形超限 | 高阶曲率、器件模型、不同电阻 TC | 增加温度点，比较曲率补偿开关 |
| 低 VDD 出现拐点 | 运放输入范围、晶体管压头、电流镜退出工作区 | 细扫 VDD，观察 IDD 与内部节点 |
| 偶发为 0 或很低 | 启动失败、test mode 时序、残余电荷 | 不同 ramp/off-time/温度重复上电 |
| 数字活动时纹波变大 | PSRR、地弹、substrate coupling、buffer 不足 | 改变数字负载频率，频谱/相关测量 |
| 封装后整体偏移 | 封装应力、焊接应力、板弯曲 | wafer/package/board 三阶段相关性 |
| 噪声随外接电容异常变化 | buffer 稳定性、测量带宽、寄生振荡 | 示波器+频谱，扫描 C/ESR |

### 13.1 一个实用的排查顺序

确认观测节点和传递函数：core、internal reference、pad 还是系统输出？

确认测试条件：VDD、温度、负载、test mode、trim、等待时间是否一致？

判断异常类型：固定偏差、斜率、曲率、供电相关、动态噪声还是启动失败？

用一个变量扫描验证因果：只扫温度、只扫 VDD、只改 ramp、只改 trim 码。

直接测与间接测做相关性，排除 MUX、PAD 和测量链路。

最后再进入版图、器件和工艺分析，避免在测试假象上过度推理。

> 对 AE 最有价值的能力：不是看到 VREF 超限就说“带隙坏了”，而是能把异常分类，并设计一个最小实验把测试问题、启动问题、trim 问题和核心温漂问题分开。

## 15. 常见误区

| 误区 | 更准确的理解 |
| --- | --- |
| 带隙基准就是固定 1.2 V | 经典核心约为 1.2 V；真实对外参考可能被缩放，且仍有误差、温漂和噪声。 |
| 25°C 测准了就说明 BGR 没问题 | 一点 trim 可能只修正截距；斜率、曲率、低压压头和启动仍可能失败。 |
| VDD 不变就不用测 PSRR | 系统电源存在纹波和数字扰动；DC line regulation 与 AC PSRR 不等价。 |
| OTP 读回正确就代表 trim 成功 | 数字码正确不保证模拟开关、码权和最终 VREF 正确；必须 post-trim 复测。 |
| ADC 码偏了就是 VREF 偏了 | ADC 码还受输入源、前端增益、ADC 本体与共模路径影响，需建立误差模型。 |
| 多取几次平均就能测噪声 | 平均只能降低某些随机误差；带宽、PSD、低频漂移和仪器底噪必须定义。 |
| 温漂只需要测两点 | 两点只能看端点斜率，可能完全漏掉中间的弓形或 S 形曲率。 |

## 16. 一页速记

> 定义：带隙基准利用 CTAT 的 VBE 与 PTAT 的 ΔVBE 加权相加，生成对温度和电源相对稳定的参考电压/电流。

> **公式**：`ΔVBE = (kT/q)ln(N)；VREF = VBE + KΔVBE`

| 要点 | 速记 |
| --- | --- |
| 为什么约 1.2 V | VBE 与 PTAT 量一阶补偿后，合成值落在 BJT 模型的外推常数 VG0（约 1.2 V）附近 |
| 为什么重要 | 它是 ADC、LDO、传感器 AFE、OSC、POR 等模块共享的内部标尺 |
| 为什么要测 | PVT、失配、曲率、启动、封装应力、trim 和测试 MUX 都会造成偏差 |
| 测试顺序 | 先确认节点/传递函数，再测初始精度、温漂、供电、负载、噪声、启动和 trim |
| 最危险误判 | 把 VREF pad 当作 VBG core；把一点精度当作全温精度；把测试链路问题当 DUT 问题 |

### 16.1 复习问答

Q1：为什么不能只用一个二极管压降做参考？

答：因为 VBE 对温度、电流和工艺都很敏感，单独使用时温漂很大。

Q2：PTAT 是怎样产生的？

答：用两个不同电流密度的 BJT 得到 ΔVBE = (kT/q)ln(N)，它与绝对温度成正比。

Q3：为什么温漂曲线还有弓形？

答：一阶斜率被抵消，但 VBE、电阻和其他器件仍存在高阶温度项。

Q4：一点 trim 能修正什么？

答：主要修正某一温度点的初始偏差，通常不能同时保证温度斜率和曲率。

Q5：line regulation 与 PSRR 区别？

答：前者看 DC/慢速 VDD 变化，后者看不同频率的 AC 纹波耦合。

Q6：为什么启动要扫 ramp rate？

答：自偏置回路可能停在零电流假稳态，上电斜率和残余电荷会改变启动路径。

Q7：没有 VREF PAD 怎样测？

答：通过 ADC 比值、比较器阈值、LDO/POR 输出等已知传递函数间接推断。

Q8：温漂为什么不能只测高低两点？

答：端点可能恰好接近，而中间出现明显曲率峰值。

Q9：如何判断异常来自测试机？

答：使用 golden unit、site swap、Kelvin sense、空载校验和不同仪器交叉验证。

Q10：对传感器芯片最关键的思路？

答：画出 VREF 的所有下游路径，判断误差在激励、AFE、ADC 和数字补偿中如何传播。
